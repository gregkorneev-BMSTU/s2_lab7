# Лабораторная работа №7

## Тема

Модификация контейнера из семинара 5 с заменой внутренней структуры хранения данных при сохранении внешнего интерфейса.

## Описание проекта

Проект представляет собой банковскую систему на C++. В программе реализованы:

- банк и отделения;
- клиенты;
- счета разных типов;
- банковская карта;
- переводы между счетами;
- история операций;
- демонстрация полиморфизма.

Финальная версия лабораторной работы выполнена на основе финальной версии семинара 5. Из нее в проект перенесены актуальная демонстрационная программа и итоговая прикладная логика, а затем поверх этой версии выполнена доработка контейнера для лабораторной 7.

## Суть лабораторной работы

В семинаре 5 контейнер `MyContainer<T>` был реализован на базе обычного динамического массива. В лабораторной работе 7 этот контейнер модифицирован: внутренняя структура хранения заменена на сегментированный динамический массив, при этом внешний интерфейс контейнера сохранен.

Это значит, что пользовательский код не пришлось переписывать: банковская система продолжает использовать те же методы контейнера, но внутри данные теперь организованы иначе.

## Что реализовано

- собственный шаблонный контейнер `MyContainer<T>`;
- новый внутренний способ хранения данных;
- ручное управление памятью;
- поддержка добавления и удаления элементов;
- доступ по индексу;
- поиск элементов;
- собственные итераторы;
- совместимость со стандартными алгоритмами `std::find` и `std::sort`;
- использование контейнера в реальной прикладной модели.

## Выбранная внутренняя структура

Для лабораторной работы выбран собственный вариант: сегментированный динамический массив.

### Как это устроено

Элементы контейнера хранятся не в одном сплошном массиве, а в нескольких блоках фиксированного размера. Для доступа к элементу по индексу вычисляются:

- номер блока;
- позиция элемента внутри блока.

При росте контейнера выделяются новые блоки, поэтому нет необходимости каждый раз переносить все элементы в единый новый буфер.

## Почему выбран именно этот вариант

- сохраняется привычный интерфейс контейнера;
- сохраняется возможность использовать `push_back`, `emplace_back`, `removeAt`, `erase`, `operator[]`, `begin()` и `end()`;
- контейнер остается совместимым со `std::find` и `std::sort`;
- структура хорошо подходит для уже готовой банковской модели;
- можно показать собственный вариант внутренней организации данных, как требуется в задании.

## Преимущества и недостатки

### Преимущества относительно обычного массива

- память можно наращивать блоками;
- не требуется переносить все элементы при каждом увеличении емкости;
- внешний интерфейс контейнера остается таким же удобным.

### Недостатки относительно обычного массива

- элементы не лежат в одном непрерывном участке памяти;
- доступ по индексу реализуется чуть сложнее;
- код контейнера становится сложнее по сравнению с базовой реализацией на одном массиве.

### Сравнение с другими возможными вариантами

- по сравнению со связанным списком сохраняется быстрый доступ по индексу;
- по сравнению с кучей сохраняется последовательная работа с элементами;
- по сравнению с деревом не требуется менять модель использования контейнера;
- по сравнению с отсортированными структурами сохраняется естественное поведение `push_back` и существующая логика банковской системы.

## Где используется контейнер

Контейнер встроен в предметную модель проекта:

- `Bank` хранит отделения в `MyContainer<Department>`;
- `Bank` хранит клиентов в `MyContainer<Client*>`;
- `Client` хранит счета клиента в `MyContainer<Account*>`;
- `Account` хранит историю операций в `MyContainer<Transaction>`;
- в демонстрации полиморфизма используется `MyContainer<std::unique_ptr<Account>>`.

Таким образом, контейнер применяется не в изолированном учебном примере, а внутри всей архитектуры программы.

## Что показывает программа

После запуска программа демонстрирует:

- создание банка, отделений и клиентов;
- открытие счетов разных типов;
- пополнение, снятие, переводы и оплату картой;
- работу контейнера на пользовательских типах;
- поиск через `std::find`;
- сортировку через `std::sort`;
- удаление элементов из контейнера;
- вывод истории операций;
- работу с простым типом `int`;
- пример полиморфизма.

Финальная версия `main.cpp` также содержит улучшенный вывод таблиц и корректное выравнивание UTF-8 строк в консоли.

## Структура проекта

- `include/` — заголовочные файлы классов;
- `src/` — исходные файлы программы;
- `task/` — методические материалы семинаров;
- `build/` — каталог сборки CMake;
- `CMakeLists.txt` — конфигурация проекта;
- `README.md` — описание проекта.

## Ключевые файлы

- `include/MyContainer.h` — финальная версия контейнера для лабораторной 7;
- `src/main.cpp` — финальная демонстрационная программа на базе семинара 5;
- `include/Bank.h`, `src/Bank.cpp` — логика банка и хранение отделений и клиентов;
- `include/Client.h`, `src/Client.cpp` — логика клиента и хранение счетов;
- `include/Account.h`, `src/Account.cpp` — базовый счет и история операций.

## Сборка и запуск
```bash
rm -rf build && cmake -S . -B build && cmake --build build && ./build/project_bank
```
